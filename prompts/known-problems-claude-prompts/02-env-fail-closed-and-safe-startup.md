# Claude Prompt 02: 환경변수 fail-open 제거 및 안전한 서버 시작 보장

## 목적

`known_problems.md`의 아래 세 문제를 함께 해결하기 위한 Claude용 실행 프롬프트다.

- `F-05` `docker-compose.yml` 의 하드코딩 DB 비밀번호 폴백
- `F-06` `CORS_ALLOWED_ORIGINS` 빈 값 시 서버가 애매하게 기동
- `F-07` `GOOGLE_CLIENT_IDS` 빈 값 시 OAuth 검증 동작 불명확

이 묶음은 "필수 설정이 비어 있으면 조용히 잘못 동작하지 말고, 명시적으로 실패하거나 경고해야 한다"는 같은 원칙으로 묶는다.

## Claude에 바로 넣는 프롬프트

```md
다음 문제들은 모두 "환경변수 누락 시 조용히 잘못 동작하는 상태"를 고치는 작업이다.

해결 대상:
1. `docker-compose.yml` 에 하드코딩된 DB 비밀번호 폴백 제거
2. `CORS_ALLOWED_ORIGINS` 가 비어 있을 때 서버가 애매하게 뜨지 않도록 처리
3. `GOOGLE_CLIENT_IDS` 가 비어 있을 때 OAuth 검증이 모호하지 않도록 처리

관련 파일:
- `docker-compose.yml`
- `apps/api/src/config/env.ts`
- `apps/api/src/plugins/cors.ts`
- `apps/api/src/modules/auth/auth.routes.ts`

문제 배경:
- 보안 관련 설정은 기본값으로 fail-open 하면 안 된다.
- 필수 설정이 비어 있는데 서버가 떠버리면, 배포 후에야 문제를 발견하게 된다.
- 현재 상태는 "실패해야 할 상황이 조용히 통과" 하거나 "동작이 라이브러리 구현에 의존" 하는 문제가 있다.

작업 원칙:
- 개발 편의보다 배포 안전성을 우선하라.
- 필수 환경변수가 비어 있으면 명시적으로 실패하거나, 최소한 운영자가 바로 알 수 있게 하라.
- 기존 코드 스타일을 유지하라.
- 필요한 최소 수정만 하라.
- 문제 해결 범위를 넘는 인증/설정 구조 개편은 하지 마라.

완료 조건:
- `POSTGRES_PASSWORD` 는 `.env` 없이 안전하게 실패한다.
- `CORS_ALLOWED_ORIGINS` 가 비어 있으면 서버가 경고 없이 애매하게 동작하지 않는다.
- `GOOGLE_CLIENT_IDS` 가 비어 있으면 인증 로직이 모호하게 동작하지 않는다.
- 어떤 설정이 필수가 되었는지 운영자가 이해할 수 있게 정리한다.

출력 형식:
1. 변경 파일
2. 변경 전 위험
3. 변경 후 동작
4. 운영자가 알아야 할 환경변수 요구사항
5. 검증 방법

가능하면 실제 코드 수정안 또는 패치 수준의 예시를 포함해라.
```
