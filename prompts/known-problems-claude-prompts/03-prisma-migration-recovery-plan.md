# Claude Prompt 03: Prisma 마이그레이션 누락 복구 계획 수립

## 목적

`known_problems.md`의 `F-01` 은 운영 DB 에 영향을 줄 수 있는 고위험 작업이다.  
이 프롬프트는 Claude에게 곧바로 수정안을 쓰게 하는 대신, 먼저 사람이 승인 가능한 복구 계획을 작성하게 한다.

## Claude에 바로 넣는 프롬프트

```md
Prisma 마이그레이션 누락 문제를 해결해야 하는데, 이 작업은 운영 DB 에 영향을 줄 수 있으므로 바로 코드 수정부터 하지 말고 먼저 안전한 실행 계획을 작성해줘.

문제:
- `apps/api/prisma/migrations/` 가 git 에 커밋되지 않아 신규 환경에서 `prisma migrate deploy` 가 실질적으로 동작하지 않는다.
- 기존 DB 에는 이미 일부 스키마가 있을 수 있다.
- 잘못된 초기 마이그레이션 생성은 운영 스키마와 충돌할 수 있다.

관련 경로:
- `apps/api/prisma/schema.prisma`
- `apps/api/prisma/migrations/`
- `apps/api/Dockerfile`
- 배포 시 `migrate deploy` 를 호출하는 경로 전반

해야 할 일:
- 현재 상황에서 가능한 마이그레이션 복구 전략을 비교해라.
- "완전 신규 DB" 와 "기존 DB 가 이미 존재하는 경우" 를 분리해서 설명해라.
- 각 전략의 위험, 장점, 선행 확인사항을 정리해라.
- 그 다음 가장 안전한 추천 전략을 제안해라.
- 실제 적용 순서를 단계별로 써라.
- 사람이 승인하기 전까지는 공격적인 자동 수정안을 내지 마라.

작업 원칙:
- 데이터 손상 가능성을 가장 먼저 고려하라.
- 운영 DB 기준으로 보수적으로 접근하라.
- 추정이 필요한 부분은 추정이라고 명시하라.
- 지금 단계에서는 "수정 계획" 과 "검증 계획" 이 핵심이다.

출력 형식:
1. 현재 문제 요약
2. 가능한 복구 전략 비교
3. 추천 전략
4. 실행 전 확인사항
5. 실제 적용 순서
6. 잘못 적용했을 때의 위험
7. 승인 후 다음 프롬프트에서 바로 쓸 수 있는 실행 지시문 초안
```
